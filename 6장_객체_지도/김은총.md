## 6장. 객체 지도(ㅁㅣㅇㅗㅏㄴ)

1. 기능적이고 해결책 지향적인 접근법보다 **구조적이고 문제 지향적인 접근법**
- 추상화된 모델
- 다양한 목적을 위해 재사용 가능
- 새로운 목적도 만족

**기능**적 요구사항 < 안정적인 **구조**
책임-주도 설계의 본질

2. 설계
미래의 변경에 대비하는 소프트웨어 설계법
= 변경되는 요구사항에 유동적으로 반응
= 변경을 수용하기 쉬운 설계
= 객체지향 접근법

3. 도메인 모델
- 도메인: 사용자가 프로그램을 사용하는 대상 분야
- 사용자가 프로그램을 사용하는 영역을 단순화, 추상화한 구조 형태

4. 표현적 차이: 은유
